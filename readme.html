Description

org.eclipse.releng.eclipsebuilder contains Ant scripts that use the java and non-java resources in org.eclipse.pde.build to build all shipped components of Eclipse: SDK, platform jdt, examples, tests and team extras. A convenience script, org.eclipse.releng.eclipsebuilder/build.xml, is provided to build any specified component with default settings of necessary Ant properties.

 

Prerequisites

** work in progress **


1. cvs executable installed and on system path variable.
2. 1.3.1 level jre installed and jre/bin directory added to system path variable.
3. 1.4 level JDK installed (for generating javadoc with working links)
4. (on non Windows systems only) - a 1.3.1 level windows java runtime environment.
5. Info-Zip zip and unzip executables on system path.
6. (for Mac OSX only) tar and gzip
7. org.eclipse.releng.basebuilder and org.eclipse.releng.eclipsebuilder checked out into a directory

cd to org.eclipse.releng.eclipsebuilder directory and execute the following command:

java -cp ../org.eclipse.releng.basebuilder/startup.jar org.eclipse.core.launcher.Main -application org.eclipse.ant.core.antRunner -buildfile build.xml -Dcomponent=sdk | platform | jdt | sdk.examples | sdk.tests [-Dconfigs=<configuration list>] [-Djavacfailonerror=true |false ] [-DjavacVerbose=true | false][-Dzipargs=<arguments to zip executable>] [ -Dbootclasspath=<absolute path to rt.jar to compile against>]

Parameters (Ant properties)

Name Description
buildDirectory The absolute path to a working directory where the source for the build will be checked, where scripts will be generated and where build results will be available.
buildId The build name. Default set to "build".
buildLabel Refers to the name of the directory which will contain the results of the build. Default set to ${buildType}.${buildId}, ie."I.build". This directoryn will be created inside the location specified by the ${buildDirectory} property.
baseLocation A directory separate from buildDirectory which contains pre-built plugins and features against which to compile.
buildType

Letters I, N, S, R or M are used in Eclipse builds to identify builds as being one of the following:

I - Integration
N - Nightly
S - Stable
R - Release
M - Maintenance

Default set to I.
collPlace The directory relative to ${buildLabel} in which built features and plugins are gathered. Default set to "eclipse".
collBase The directory relative to ${buildLabel} in which to execute zip of the ${collPlace}directory and configuration specific root files.
configs

An ampersand separated list of configurations to build where a configuration is specified as <os>,<ws>,<arch>.
ie.configs=win32,win32,x86 & linux, motif, x86 & linux, gtk, x86
bootclasspath Sets the value for the attribute "bootclasspath" in calls to the Ant <javac> task in a plugins' build.xml. Default set to ${java.home}/lib/rt.jar.
javacFailOnError Sets the value for the attribute "failonerror" in calls to the Ant <javac> task plugins' build.xml. Build will continue even if there are compilation errors when this is set to false.
javacDebugInfo Sets the value for the attribute "debug" in calls to the Ant <javac> task to the value set for this property. Default set to on.
javacVerbose Sets the value for the attribute "verbose" in calls to the Ant <javac> task to the value set for this property. Default set to true.
javacSource Sets the value for the attribute "source" in calls to the Ant <javac> task to the value set for this property. Default set to 1.3.
javacTarget Sets the value for the attribute "target" in calls to the Ant <javac> task to the value set for this property. Default set to 1.1.
mapVersionTag

Sets the tag attribute in a call to the Ant <cvs> task to check out org.eclipse.releng, the map file project on dev.eclipse.org used in the build.

Can be set to a specific tag to download the map files used for a previous integration build. Typically, these tags are in the form "v<date and timestamp of build>" for example "v200307110800" will checkout the map files used to run the integration at the specified date/time. This mechanism will only work with builds > 20030701.
zipargs Arguments to send to the zip executable. ie. zipargs=-y on Linux preserves symbolic links.
baseos,basews,basearch The os, ws and arch values of a pre-built eclipse SDK configuration which is being compiled against. See list of supported values in the table below.
timestamp A timestamp used in about.mappings files and to tag the org.eclipse.releng project on dev.eclipse.org (only available to folks with commit rights on this project.)
