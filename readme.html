Prerequisites

** work in progress **


1. cvs executable installed and on system path variable.
2. 1.3.1 level jre installed and jre/bin directory added to system path variable.
3. 1.4 level JDK installed (for generating javadoc with working links)
4. (on non Windows systems only) - a 1.3.1 level windows java runtime environment.
5. Info-Zip zip and unzip executables on system path.
6. (for Mac OSX only) tar and gzip
7. org.eclipse.releng.basebuilder and org.eclipse.releng.eclipsebuilder checked out into a directory

cd to org.eclipse.releng.eclipsebuilder directory and execute the following command:

java -cp ../org.eclipse.releng.basebuilder/startup.jar org.eclipse.core.launcher.Main -application org.eclipse.ant.core.antRunner -buildfile build.xml -Dcomponent=sdk | platform | jdt | sdk.examples | sdk.tests [-Dconfigs=<configuration list>] [javacfailonerror=true |false ] [javacVerbose=true | false][zipargs=<arguments to zip executable>] [ -Dbootclasspath=<absolute path to rt.jar to compile against>]

 

examples:

java -cp ../org.eclipse.releng.basebuilder/startup.jar org.eclipse.core.launcher.Main -application org.eclipse.ant.core.antRunner -buildfile build.xml -Dcomponent=sdk | platform | jdt | sdk.examples | sdk.tests | team.extras

Ant properties

Property Description
mapVersionTag

The tag to use when checking out the map file project org.eclipse.releng,

ie "cvs -d :pserver:anonymous@dev.eclipse.org:/home/eclipse co -r ${mapVersionTag} org.eclipse.releng"

Can be set to a specific tag to download the map files used for a previous integration build. Typically, these tags are in the form "v<date and timestamp of build>" for example "v200307110800" will checkout the map files used to run the integration at the specified date/time. This mechanism will only work with builds > 20030701.
buildDirectory The absolute path to a working directory where the source for the build will be checked, where scripts will be generated and where build results will be available.
buildType

Letters I, N, S, R or M are used in Eclipse builds to identify builds as being one of the following:

I - Integration
N - Nightly
S - Stable
R - Release
M - Maintenance

Default set to I.
buildId Refers to the build name. Default set to "build".
buildLabel Refers to the name of the directory which will contain the results of the build. Default set to ${buildType}.${buildId}, ie."I.build". This directoryn will be created inside the location specified by the ${buildDirectory} property.
baseLocation A directory separate from buildDirectory whic contains pre-built plugins and features against which to compile.
collPlace The directory relative to ${buildLabel} in which built features and plugins are gathered. Default set to "eclipse".
collBase The directory relative to ${buildLabel} in which to execute zip of the ${collPlace}directory and configuration specific root files.
configs

An ampersand separated list of configurations to build where a configuration is specified as <os>,<ws>,<arch>, ie. win32,win32,x86.
Two configurations can be specified as follows: configs=win32,win32,x86 & linux, motif, x86
bootclasspath Sets the attribute "bootclasspath" in calls to the Ant <javac> task to the value set for this property. The absolute path to the java libraries against which to compile (typically the jre/lib/rt.jar of a jre or jdk).
javacFailOnError Sets the attribute "failonerror" in calls to the Ant <javac> task to the value set for this property. Build will continue even if there are compilation errors when this is set to false.
javacDebugInfo Sets the attribute "debug" in calls to the Ant <javac> task to the value set for this property. Default set to on.
javacVerbose Sets the attribute "verbose" in calls to the Ant <javac> task to the value set for this property. Default set to true.
javacSource Sets the attribute "source" in calls to the Ant <javac> task to the value set for this property. Default set to 1.3.
javacTarget Sets the attribute "target" in calls to the Ant <javac> task to the value set for this property. Default set to 1.1.
zipargs Arguments to send to the zip executable. ie. zipargs=-y on Linux requests zip to preserve symbolic links.
baseos,basews,basearch The os, ws and arch values of a pre-built eclipse SDK configuration which is being compiled against.
timestamp A timestamp used in about.mappings files and to tag the org.eclipse.releng project on dev.eclipse.org (only available to folks with commit rights on this project.)
